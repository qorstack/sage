AGENTS.md — The Protocol

AGENTS.md คือ Sage ทั้งหมด — ไม่มี server, ไม่มี Python, ไม่มี build step. AI agent ใดก็ตามที่อ่านไฟล์นี้ก่อนทำงานจะ follow the protocol อัตโนมัติ

Pipeline §1–4 (Before code)

  1. Become the role — เลือก senior lens ที่เหมาะสม (dev, architect, qa, designer, …) แล้วเปิด agents/sage/roles/role-<lens>.md
  2. Read the knowledge — เปิด agents/sage/<domain>/rules.md และ decisions/*.md ที่เกี่ยวข้อง
  3. Find reusable assets — อ่าน source file ของ service/util/component ที่ rules ชี้ไว้ก่อนเขียนโค้ดที่ใช้มัน
  4. Assess & plan — ประเมิน verdict (proceed / warn / ask / reject), แบ่งงานเป็น [parallel] / [sequential], assign effort tier

After code (mandatory)

  1. Capture knowledge → agents/sage/<domain>/decisions/<slug>.md
  2. Output the mandatory summary block (role · domain · risk · done · decisions · validated · knowledge)
📌
Single source of truth. ห้าม fork wording ของ protocol เข้าไปใน integration files — integration files แค่ pointer มาที่ AGENTS.md

agents/sage/ — Knowledge Store

Knowledge store เติบโตได้สองทาง: human เขียนเอง หรือ AI เขียนให้หลังทำงาน. ทุก entry เป็น YAML frontmatter + Markdown body — 1 idea ต่อ 1 ไฟล์

Directory layout

agents/sage/
  index.md                        # table of contents
  roles/
    role-dev.md                   # Senior Developer persona
    role-architect.md             # created on first use
    role-writer.md                # created by sage-docs
  <domain>/                       # e.g. billing, auth, frontend
    rules.md                      # standing rules for this domain
    decisions/
      <slug>.md                   # one team decision per file
    skills/
      <slug>.md                   # best practice / how-to playbook
  docs/
    docs-style.css                # CSS template — inlined by sage-docs

Data Flow — How It All Connects

  • 1
    Install integration (one-time)
    cp -r integrations/.claude . — copy adapter files เข้า repo ของทีม. Codex และ Gemini CLI ไม่ต้อง copy เพราะอ่าน AGENTS.md ได้เลย
  • 2
    Bootstrap knowledge (one-time)
    รัน /sage-learning ครั้งแรก — AI scan codebase แล้วเขียน rules, decisions, skills เข้า agents/sage/ ใน repo ของทีม
  • 3
    AI อ่าน AGENTS.md ก่อนทุก task
    Adapter บอก AI ให้อ่าน AGENTS.md → AI run pipeline §1–4: pick role → read domain knowledge → find reusable assets → assess & plan
  • 4
    AI เขียนโค้ด + capture knowledge
    หลังเขียนโค้ด AI เขียน pattern ที่เรียนรู้ได้กลับเข้า agents/sage/<domain>/decisions/ ทุก run — ความรู้ของทีมเพิ่มขึ้นโดยอัตโนมัติ
  • 5
    Generate human-readable docs (on demand)
    รัน /sage-docs → AI วิเคราะห์ source material → classify doc type → generate inline SVG diagram → write docs/<slug>.html with CSS inlined

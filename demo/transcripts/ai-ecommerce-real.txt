- Loves      - interfaces that are fast, readable, and obviously correct at a glance.
- Good at    - semantic HTML, CSS layout (grid/flex), vanilla JS, asset reuse,
               accessible markup, and shipping static pages that need zero build step.
- Team needs - production-ready UI without framework overhead on marketing / landing pages.
- Worth it   - pages that load fast, look right on mobile, and any dev can edit without tooling.

## How I work
- Start from the layout skeleton, then fill in content, then add visual polish.
- Reuse existing assets (logos, colors, fonts) before introducing anything new.
- No framework unless the team has already adopted one in this domain.
- Accessibility-first: semantic tags, alt text, focus states.
- Follow the domain's `rules.md` when a matching Sage domain exists.

## Decision

Static landing pages for product / ecommerce use a four-section layout:
**hero → collection grid → trust strip → checkout CTA**, all in one HTML file
with inline `<style>`. No framework, no build step.

## Rationale

- This repo has no frontend build pipeline. A single self-contained HTML file
  is the only format any contributor can edit and preview without tooling.
- The four-section order matches the buyer journey: awareness → browse →
  trust → convert.

## Conventions observed

| Concern       | Convention |
| ------------- | ---------- |
| Color tokens  | CSS custom properties on `:root` — `--bg`, `--text`, `--accent`, `--muted` |
| Accent color  | `#d4a853` (gold) — matches existing landing theme |
| Typography    | `Georgia` serif; `clamp()` for responsive headings |
| Layout        | CSS Grid with `auto-fill / minmax` for product grids; Flexbox for nav/trust |
| Asset reuse   | `logo.png` for favicon + mobile; `logo-full.png` for hero lockup |
| CTA style     | Accent-background pill, uppercase letter-spaced label, `:hover { opacity:.85 }` |
| No JS         | Interactions (hover lift on cards) handled with CSS `transition` only |

## Reusable assets

- `landing/logo.png` — 32 px nav / favicon
- `landing/logo-full.png` — hero wordmark / lockup

            minmax so it reflows gracefully at any viewport width without media queries.

            new assets.

            Card hover lift and CTA opacity fade both work without JS. Logo loads
